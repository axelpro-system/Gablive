# E2E Testing Implementation Plan

## Overview

Set up Playwright E2E tests for the webinar-saas project following the hybrid approach: mock Supabase for public flows, real staging for auth/dashboard.

**Design doc:** `thoughts/shared/designs/2026-07-22-e2e-testing-design.md`

---

## Task DAG

```
Phase 1 ──┬── 1.1 Install Playwright
           └── 1.2 Create playwright.config.ts

Phase 2 ──┬── 2.1 Create directory structure
           ├── 2.2 fixtures/webinar.ts (data factories)
           ├── 2.3 fixtures/supabase.ts (route interceptors)
           ├── 2.4 pages/RegistrationPage.ts
           ├── 2.5 pages/WebinarRoomPage.ts
           ├── 2.6 pages/WaitRoomPage.ts
           └── 2.7 pages/LoginPage.ts

Phase 4 ──┬── 4.1 Update package.json scripts
           └── 4.2 Update .gitignore + verify build
```

## Tasks

### Phase 1: Setup

#### 1.1 Install Playwright
- Run `npm init playwright@latest -- --ct` or install `@playwright/test` manually
- Install browsers: `npx playwright install chromium`
- **Verify:** `npx playwright --version` succeeds

#### 1.2 Create playwright.config.ts
- File: `tests/e2e/playwright.config.ts`
- Projects: `critical-flows` (mock-based, retries=1), `chromium` (retries=2)
- webServer: `npm run dev` on port 3000, reuseExistingServer=true
- Use: baseURL=http://localhost:3000, trace=on-first-retry, screenshot=only-on-failure, video=retain-on-failure
- testDir: `./specs`, fullyParallel: true
- **Verify:** `npx playwright test --config=tests/e2e/playwright.config.ts --list` lists zero tests

### Phase 2: Test Infrastructure

#### 2.1 Create directory structure
- `tests/e2e/fixtures/` — supabase.ts, webinar.ts, auth.ts
- `tests/e2e/pages/` — RegistrationPage.ts, WebinarRoomPage.ts, WaitRoomPage.ts, LoginPage.ts, DashboardPage.ts
- `tests/e2e/specs/critical/` — test files
- **Verify:** All directories exist, each has `__init__` placeholder if needed

#### 2.2 fixtures/webinar.ts — Data factories
- Types: `WebinarData`, `AudienceConfigData`, `CtaConfigData`, `RegistrationPageData`, `ChatMessageData`
- `createWebinar(overrides?)`: Returns full webinar with nested audience_configs, cta_configs, registration_pages, chat_messages
- `createAudienceConfig(overrides?)`: Returns `{ mode: 'dynamic'|'fixed'|'none', current_count: number }` as object, NOT array
- `createRegitrationPage(overrides?)`: Returns page with blocks and theme
- **CRITICAL:** audience_configs typed as `object | null`, NOT `object[]` — this is the bug we fixed
- **Verify:** TypeScript compiles without errors

#### 2.3 fixtures/supabase.ts — Route interceptors
- `mockWebinarPage(page, data)`: Registers page.route() for:
  - PostgREST webinar query (`...rest/v1/webinars?...`)
  - PostgREST audience_configs (`...rest/v1/audience_configs?...`)
  - Chat messages (`...rest/v1/chat_messages?...`)
  - CTA configs (`...rest/v1/cta_configs?...`)
  - Registration check (`...rest/v1/registrations?...`)
  - YouTube/Vimeo embeds (`*youtube.com*`, `*vimeo.com*` → 204)
- `mockRegistrationPage(page, data)`: Same pattern for registration flow
- Each route logs calls for debugging
- Unmatched routes fail the test (not silent pass)
- **CRITICAL:** audience_configs returns as object, not wrapped in array
- **Verify:** Simple test that navigates and checks mock data appears

#### 2.4 pages/RegistrationPage.ts
- `goto(slug)`, `fillForm({name, email, phone})`, `submit()`, `isSuccessVisible()`
- Locators: form inputs (data-testid), submit button, success message, error messages
- **Verify:** Imports and compiles

#### 2.5 pages/WebinarRoomPage.ts
- `waitForPlayer()`, `getAudienceCount()`, `getChatMessages()`, `clickCTA()`
- Locators: video container, audience counter, chat messages list, CTA banner
- **Verify:** Imports and compiles

#### 2.6 pages/WaitRoomPage.ts
- `waitForCountdown()`, `getAudienceCount()`
- Locators: countdown timer, audience display
- **Verify:** Imports and compiles

#### 2.7 pages/LoginPage.ts
- `goto()`, `login(email, password)`, `isLoggedIn()`
- Locators: email input, password input, submit button
- **Verify:** Imports and compiles

### Phase 3: Critical Tests

#### 3.1 specs/critical/audience-counter.spec.ts
Three test cases:
1. **Dynamic mode**: mock returns `{ mode: 'dynamic', current_count: 42 }` → counter visible with "42"
2. **Fixed mode**: mock returns `{ mode: 'fixed', current_count: 15 }` → counter visible with "15"
3. **None mode**: mock returns `{ mode: 'none' }` → counter hidden

Each test uses `mockWebinarPage` with appropriate data, then navigates to `/room/test-webinar`
- **Verify:** `npx playwright test specs/critical/audience-counter.spec.ts --config=playwright.config.ts` passes

#### 3.2 specs/critical/registration.spec.ts
Four test cases:
1. **Successful registration**: fill form, submit, assert success message visible
2. **Validation error**: submit empty form, assert error shown
3. **Invalid email**: fill with bad email, assert validation error
4. **Already registered**: mock registration check returns existing record, assert "already registered" error

Uses `mockRegistrationPage`
- **Verify:** Test passes

#### 3.3 specs/critical/chat-messages.spec.ts
Two test cases:
1. **Chat renders**: assert chat container visible with messages
2. **Simulated messages at timestamps**: mock messages with specific `time_seconds`, verify they appear at correct player time

Uses `mockWebinarPage` with chat_messages data
- **Verify:** Test passes

#### 3.4 specs/critical/cta-banner.spec.ts
Two test cases:
1. **CTA appears at correct time**: mock cta_configs with `pitch_start_seconds`, simulate player at that time, assert banner visible
2. **Pricing display**: original_price and sale_price formatted correctly

Uses `mockWebinarPage` with cta_configs
- **Verify:** Test passes
